Status Posting System

Special Features

I used buttons for the hyperlinks because I thought it would look nicer. All pages are styled using the same style sheet and have a nice background picture. The database has been populated with a number of statuses with the keyword burger in it. This is a good search word to used to test in the search function. The date feature for posting uses a nice calendar to help choose a date and is prepopulated with todays date.

Troubles

My main troubles was with SQL statements. This was just because I was out of practice and I had never used it in PHP before. Sometimes I would have symbols in the wrong places or the SQL command just wasn't right. I also had some small problems with parts of the style sheet overriding other parts.

Future Improvements

I could see putting the buttons in a column on the left side of the page would look nicer. Another improvement would be to automatically generate the status code for the user. This way they wouldn't have to guess one. Trying to guess an available status code could become very problematic as more and more statuses are succesfully loaded into the database. Another thing to improve on would be to improve the readability of the text. Either by changing the background or the text color.
